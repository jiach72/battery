# 电池健康管理平台 - 生产交付清单

> 目标：把当前项目从“可演示/可联调”推进到“可上线/可交付”。

## 1. 交付门槛

上线前必须同时满足：

- 前端、后端、算法服务可独立构建
- Docker Compose 生产编排可一键启动
- 核心接口有真实联调结果，不依赖 demo fallback
- 设备接入、告警、日志、备份、回滚都有明确流程
- 客户侧权限、账号、证书、域名、端口已确认

## 2. P0 必做

### 2.1 部署闭环

- 补齐 `docker-compose.prod.yml`
- 增加前端生产镜像
- 让 Nginx 统一承载前端静态资源和 `/api` 反代
- 明确每个服务的健康检查、重启策略、资源限制

### 2.2 配置收口

- 所有 `change_me` 占位符改为部署时注入
- 关闭生产环境 demo 登录和 demo 数据
- JWT、MQTT、Kafka、数据库、算法内网密钥改为 Secret 管理
- 删除默认回退值，避免“误启动即生产”

### 2.3 数据接入

- 明确站点、单元、簇、单体、测点主数据
- 固化遥测 JSON 规范
- 固化 MQTT topic 命名
- 固化 InfluxDB measurement/tag/field 规范

### 2.4 安全

- 生产环境 token 策略收口
- RBAC 权限矩阵落地
- 接口限流和审计日志补齐
- 设备上报链路加鉴权和签名

## 3. P1 建议

- 接入 Prometheus / Grafana / 日志平台
- Kafka 消费失败重试和死信队列完善
- 备份恢复脚本自动化
- 升级与回滚脚本自动化
- 端到端测试覆盖真实流程

## 4. P2 增强

- 多租户隔离
- 操作审计导出
- 告警闭环工单
- SSO / LDAP / 企业微信等客户集成

## 5. 验收清单

- 登录、刷新、退出正常
- 设备台账、拓扑、量测映射正常
- 实时遥测写入 InfluxDB 正常
- 告警能从设备数据触发到前端展示
- 算法调用可回写结果
- 备份与恢复可演练
- 生产环境禁用 demo fallback

## 6. 交付物

- 部署手册
- 运维手册
- 接口文档
- 设备接入协议
- 点表模板
- 验收报告
- 回滚方案
- 备份方案
